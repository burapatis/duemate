# ชุดทดสอบเบต้า DueMate

**สถานะ:** DueMate v0.7.0 — เตรียมทดสอบผู้ใช้กลุ่มเล็ก

เอกสารนี้เป็นหน้ารวมชุดทดสอบเบต้า อธิบายว่าไฟล์ต่าง ๆ ใช้อย่างไร และควรทดสอบ DueMate กับผู้ใช้จริงตามลำดับใด

---

## วัตถุประสงค์

ชุดทดสอบนี้ใช้เพื่อ:

- ตรวจว่าแอป **ใช้งานง่าย** สำหรับผู้ใช้ทั่วไป
- ตรวจว่าผู้ใช้ **เข้าใจ** การเพิ่ม / แก้ไข / ลบ / ค้นหา / ส่งออก / ตั้งเตือน
- **เก็บปัญหา** ด้าน UI, ข้อความ, และความสับสน **ก่อนเผยแพร่จริง** บน Store

## ลำดับการใช้งานที่แนะนำ

### 1) เตรียมอุปกรณ์ทดสอบ

- ติดตั้ง DueMate v0.7.0 บน Android, iPhone หรือ macOS
- ตรวจว่าแอปเปิดได้และหน้า Home แสดงปกติ
- เตรียมกระดาษหรือไฟล์สำหรับบันทึก feedback
- กำหนดเวลาทดสอบประมาณ **20–30 นาที** ต่อคn

### 2) รีเซ็ตข้อมูลในแอปก่อนทดสอบ (ถ้าจำเป็น)

- ถ้าเครื่องเคยใช้ทดสอบมาก่อน ไปที่ **ตั้งค่า** → **ล้างข้อมูลทั้งหมดในเครื่อง**
- ช่วยให้ผู้ทดสอบเริ่มจากสภาพแวดล้อมสะอาด ไม่สับสนกับข้อมูลเก่า

### 3) ให้ผู้ใช้ทำตาม REAL_USER_TEST_SCRIPT.md

- เปิด [`REAL_USER_TEST_SCRIPT.md`](REAL_USER_TEST_SCRIPT.md)
- อธิบายสั้น ๆ ว่าเป็นเวอร์ชันทดสอบ แล้วให้ลองทำตาม **7 งานหลัก**
- สังเกตและจดพฤติกรรมระหว่างทดสอบ
- ถามคำถามหลังทดสอบตามในสคริปต์

### 4) บันทึกผลใน FEEDBACK_FORM_TEMPLATE.md

- เปิด [`FEEDBACK_FORM_TEMPLATE.md`](FEEDBACK_FORM_TEMPLATE.md)
- ให้ผู้ทดสอบกรอกคะแนนและความคิดเห็น (ไม่บังคับชื่อจริง)
- บันทึกปัญหาที่พบในตาราง (ถ้ามี)

### 5) สรุปปัญหาที่พบซ้ำ

- รวบรวม feedback จากผู้ทดสอบทุกคn
- จัดกลุ่มปัญหาที่ **หลายคnพบเหมือนกัน**
- ใช้ส่วน **「สรุปผลสำหรับผู้พัฒนา」** ในแบบฟอร์ม feedback

### 6) จัดลำดับสิ่งที่ควรแก้ก่อน

- แก้ปัญหาระดับ **สูง** ก่อน (crash, ใช้งานไม่ได้, กลัวใช้ต่อ)
- จากนั้นปรับ **ข้อความ** และ **UI** ที่ทำให้สับสนซ้ำ ๆ
- พิจารณาฟีเจอร์ที่ผู้ใช้ **ขอซ้ำ** สำหรับรอบถัดไป

---

## จำนวนผู้ทดสอบที่แนะนำ

**5–10 คn** — ผู้ใช้ทั่วไปที่มีเอกสารหรือวันครบกำหนดต้องจำ  
ไม่จำเป็นต้องเป็นนักพัฒนา

---

## ข้อควรระวัง

- **ไม่ให้ผู้ใช้กรอกเลขเอกสารจริง** — ใช้ตัวอย่าง เช่น 「ใบขับขี่」 「พ.ร.บ. รถยนต์」
- **ไม่เก็บข้อมูลส่วนบุคคลเกินจำเป็น** — ไม่บังคับชื่อจริง เบอร์โทร หรืออีเมล
- **แจ้งผู้ทดสอบว่าเป็นเวอร์ชันทดสอบ** — ยังไม่ใช่เวอร์ชันเผยแพร่จริงบน Store
- **ข้อมูลเก็บในเครื่องเท่านั้น** — ไม่อัปโหลดออกจากเครื่อง

---

## เกณฑ์พร้อมไปต่อหลังทดสอบ

DueMate v0.7.0 ถือว่า **พร้อมไปขั้นถัดไป** เมื่อครบเงื่อนไขต่อไปนี้:

- [ ] **ไม่มี crash** ระหว่างทดสอบ
- [ ] ผู้ทดสอบ **ทำงานหลักได้** (เพิ่ม / แก้ไข / ลบ / ค้นหา / ส่งออก / ตั้งเตือน)
- [ ] **ข้อความสำคัญเข้าใจได้** เช่น 「เตือนล่วงหน้า」 「บันทึก」 「ล้างข้อมูล」
- [ ] **ไม่มีจุดที่ทำให้ผู้ใช้กลัวหรือกดผิดซ้ำมาก** โดยเฉพาะปุ่มลบและล้างข้อมูล
- [ ] **ปัญหาที่พบถูกบันทึกแล้ว** และจัดลำดับการแก้ไขชัดเจน
