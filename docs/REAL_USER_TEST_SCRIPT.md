# DueMate — สคริปต์ทดสอบผู้ใช้จริง (v0.9.0)

เอกสารนี้ใช้สำหรับทดสอบ DueMate กับผู้ใช้จริงกลุ่มเล็ก (ประมาณ 5–10 คน)  
เวอร์ชันนี้เป็น **เวอร์ชันทดสอบ** ยังไม่ใช่เวอร์ชันเผยแพร่จริงบน Store

---

## 1. วัตถุประสงค์การทดสอบ

- ตรวจว่าผู้ใช้ทั่วไป **ใช้งานฟีเจอร์หลักได้** โดยไม่ต้องมีคนอธิบายทุกขั้นตอน
- ตรวจว่าข้อความในแอป **อ่านเข้าใจง่าย** และปุ่มต่าง ๆ **กดได้ไม่สับสน**
- หาจุดที่ผู้ใช้ **กลัวกดผิด** หรือ **ไม่แน่ใจว่าแอปบันทึกหรือยัง**
- ยืนยันว่าแอป **ไม่ crash** และทำงานหลักครบตามที่ออกแบบไว้

---

## 2. กลุ่มผู้ทดสอบที่แนะนำ

| ลักษณะ | รายละเอียด |
|--------|------------|
| จำนวน | 5–10 คน |
| ประเภท | ผู้ใช้ทั่วไป ไม่จำเป็นต้องเป็นนักพัฒนา |
| ลักษณะที่เหมาะ | คนที่มีเอกสารหรือวันครบกำหนดต้องจำ เช่น ใบขับขี่ พ.ร.บ. รถยนต์ ประกัน สัญญาเช่า หรือเอกสารราชการ |
| อุปกรณ์ | Android หรือ macOS ที่ติดตั้ง DueMate แล้ว |

## 3. สิ่งที่ต้องเตรียมก่อนทดสอบ

### ฝั่งผู้พัฒนา / ผู้ดูแลการทดสอบ

- [ ] ติดตั้ง DueMate v0.9.0 บนเครื่องผู้ทดสอบ
- [ ] ตรวจว่าแอปเปิดได้และหน้า Home แสดงปกติ
- [ ] เตรียมเอกสารตัวอย่าง 1–2 รายการ (เช่น 「ใบขับขี่」 「พ.ร.บ. รถยนต์」) สำหรับให้ผู้ทดสอบลองกรอก
- [ ] เตรียมกระดาษหรือไฟล์โน้ตสำหรับจด feedback
- [ ] กำหนดเวลาทดสอบประมาณ **20–30 นาที** ต่อคน

### ฝั่งผู้ทดสอบ

- ไม่ต้องเตรียมบัญชีหรือสมัครสมาชิก — ข้อมูลเก็บในเครื่องเท่านั้น
- ถ้าทดสอบการแจ้งเตือน ควรอนุญาตให้แอปส่งการแจ้งเตือนเมื่อแอปถาม

---

## 4. ขั้นตอนก่อนเริ่มทดสอบ

1. เปิดแอป DueMate ให้ผู้ทดสอบ
2. บอกสั้น ๆ ว่า:
   - 「นี่เป็นแอปช่วยบันทึกและเตือนวันครบกำหนดเอกสาร」
   - 「เป็นเวอร์ชันทดสอบ ข้อมูลเก็บในเครื่องนี้เท่านั้น」
   - 「ลองใช้ตามใจได้ ถ้าติดตรงไหนบอกได้เลย」
3. **ไม่สอนทีละปุ่ม** — ให้ผู้ทดสอบลองเองก่อน (ช่วยเมื่อติดจริง ๆ เท่านั้น)
4. สังเกตและจดพฤติกรรมระหว่างทดสอบ
5. หลังครบทุกงาน ถามคำถามในหัวข้อ 6

## 5. รายการงานทดสอบ

ทำตามลำดับด้านล่าง หรือสลับได้ถ้าจำเป็น — แต่ควรครบทุกข้อ

---

### งานที่ 1: เพิ่มรายการใหม่

**เป้าหมาย:** ผู้ใช้เพิ่มเอกสารใหม่ได้ครบถ้วน

**ให้ผู้ทดสอบลอง:**
1. จากหน้า Home กด **「เพิ่มรายการใหม่」**
2. กรอกชื่อรายการ (ใช้ตัวอย่าง เช่น 「ใบขับขี่」)
3. เลือกหมวด
4. เลือกวันครบกำหนด
5. เลือกเตือนล่วงหน้า (เช่น 7 วัน)
6. กรอกหมายเหตุ (ถ้ามี)
7. กด **「บันทึก」**

**สิ่งที่สังเกต:**
- แตะช่องกรอกแล้ว keyboard ขึ้นไหม
- รู้ไหมว่าต้องเลือกวันครบกำหนดก่อนบันทึก
- เข้าใจคำว่า **「เตือนล่วงหน้า」** ไหม
- หลังบันทึก เห็นรายการใหม่บน Home ไหม

| ผล | บันทึก |
|----|--------|
| ผ่าน / ไม่ผ่าน | |
| หมายเหตุ | |

---

### งานที่ 2: แก้ไขรายการ

**เป้าหมาย:** ผู้ใช้แก้ไขรายการที่มีอยู่ได้

**ให้ผู้ทดสอบลอง:**
1. แตะรายการจากหน้า Home เพื่อเข้าหน้ารายละเอียด
2. กด **「แก้ไข」**
3. เปลี่ยนชื่อหรือวันที่หรือหมายเหตุ
4. กด **「บันทึก」**

**สิ่งที่สังเกต:**
- หาเมนูแก้ไขได้ไหม
- เข้าใจไหมว่าต้องกดบันทึกถึงจะเก็บการเปลี่ยนแปลง
- ถ้ากดกลับโดยไม่บันทึก — มีข้อความเตือนไหม (ถ้ามีการแก้ไข)

**ทดสอบเพิ่ม (ถ้ามีเวลา):**
- แก้ไขแล้วกดกลับ **โดยไม่บันทึก** — ดูว่าเข้าใจ dialog เตือนหรือไม่

| ผล | บันทึก |
|----|--------|
| ผ่าน / ไม่ผ่าน | |
| หมายเหตุ | |

---

### งานที่ 3: ลบรายการ

**เป้าหมาย:** ผู้ใช้ลบรายการได้ และเข้าใจว่าลบแล้วกู้คืนไม่ได้

**ให้ผู้ทดสอบลอง:**
1. เข้าหน้ารายละเอียดของรายการ
2. กด **「ลบ」**
3. ยืนยันการลบ

**สิ่งที่สังเกต:**
- กลัวกดปุ่มลบไหม
- เข้าใจข้อความยืนยันก่อนลบไหม
- หลังลบ รายการหายจาก Home ไหม

| ผล | บันทึก |
|----|--------|
| ผ่าน / ไม่ผ่าน | |
| หมายเหตุ | |

---

### งานที่ 4: ค้นหาและกรองรายการ

**เป้าหมาย:** ผู้ใช้หารายการที่ต้องการได้

**ให้ผู้ทดสอบลอง:**
1. จากหน้า Home กด **「ค้นหา」**
2. พิมพ์ชื่อรายการเพื่อค้นหา
3. ลองกรองตามหมวด

**สิ่งที่สังเกต:**
- หาเมนูค้นหาได้ไหม
- ผลการค้นหาตรงกับที่คาดไหม
- เข้าใจตัวเลือกกรองหมวดไหม

| ผล | บันทึก |
|----|--------|
| ผ่าน / ไม่ผ่าน | |
| หมายเหตุ | |

---

### งานที่ 5: ตั้งเตือน

**เป้าหมาย:** ผู้ใช้เข้าใจและตั้งการแจ้งเตือนได้

**ให้ผู้ทดสอบลอง:**
1. เพิ่มหรือแก้ไขรายการ แล้วเลือกวัน **「เตือนล่วงหน้า」** (เช่น 30, 15, 7, 1 วัน)
2. บันทึกรายการ
3. (ถ้ามีเวลา) ไปที่ **ตั้งค่า** → กด **「ลองส่งการแจ้งเตือน」** เพื่อตรวจว่าเครื่องอนุญาตแจ้งเตือนได้

**สิ่งที่สังเกต:**
- เข้าใจว่า 「เตือนล่วงหน้า」 หมายถึงแจ้งก่อนวันครบกำหนดไหม
- รู้ไหมว่าต้องอนุญาตการแจ้งเตือนจากระบบ
- ข้อความหลังบันทึก (เช่น 「ตั้งเตือนแล้ว」) เข้าใจไหม

| ผล | บันทึก |
|----|--------|
| ผ่าน / ไม่ผ่าน | |
| หมายเหตุ | |

### งานที่ 6: ส่งออก CSV หรือ PDF

**เป้าหมาย:** ผู้ใช้ส่งออกรายการเป็นไฟล์ได้

**ให้ผู้ทดสอบลอง:**
1. จากหน้า Home กด **「ส่งออก」**
2. เลือกรูปแบบไฟล์ (CSV หรือ PDF)
3. เลือกขอบเขตรายการ (เช่น ทั้งหมด)
4. กดปุ่มสร้างและแชร์ไฟล์

**สิ่งที่สังเกต:**
- เข้าใจความต่าง CSV กับ PDF ในระดับพอใช้งานไหม
- **PDF ภาษาไทย** (ชื่อรายการ/หมวด) อ่านได้ชัดไหม
- เห็นข้อความเตือนเรื่อง **ข้อมูลที่แชร์ออกจากเครื่อง** ก่อนกดแชร์ไหม
- ชื่อไฟล์ที่ส่งออกมีวันที่/เวลา ไม่ทับไฟล์เดิมเมื่อส่งออกซ้ำไหม
- หน้าต่างแชร์/บันทึกไฟล์เปิดได้ไหม
- สับสนกับคำว่า 「ขอบเขต」 หรือ 「scope」 ไหม

| ผล | บันทึก |
|----|--------|
| ผ่าน / ไม่ผ่าน | |
| หมายเหตุ | |

### งานที่ 7: ล้างข้อมูลทั้งหมดในเครื่อง

**เป้าหมาย:** ผู้ใช้ล้างข้อมูลได้ และเข้าใจผลที่ตามมา

> **หมายเหตุ:** ควรทำงานนี้ **ท้ายสุด** เพราะจะลบรายการทั้งหมด

**ให้ผู้ทดสอบลอง:**
1. ไปที่ **ตั้งค่า**
2. กด **「ล้างข้อมูลทั้งหมดในเครื่อง」**
3. ยืนยันการล้าง

**สิ่งที่สังเกต:**
- กลัวกดปุ่มนี้ไหม
- เข้าใจว่าล้างแล้วกู้คืนไม่ได้ไหม
- หลังล้าง รายการบน Home หายหรือกลับเป็นค่าเริ่มต้นไหม

|------|--------|--------|
| 1 | โดยรวมแล้ว อ่านข้อความในแอปง่ายไหม | |

| 3 | เข้าใจคำว่า **「เตือนล่วงหน้า」** ไหม หมายความว่าอะไร | |

## 8. เกณฑ์ผ่านเบื้องต้น

DueMate v0.9.0 ถือว่า **ผ่านการทดสอบเบื้องต้น** เมื่อครบเงื่อนไขต่อไปนี้:

- [ ] **ไม่มี crash** ระหว่างทดสอบทุกงาน
- [ ] ผู้ทดสอบอย่างน้อย **80%** ทำงานหลัก 7 ข้อได้ (อาจต้องช่วยเล็กน้อย)
- [ ] **ไม่มีปัญหาระดับสูง** ที่ทำให้ใช้งานต่อไม่ได้
- [ ] ไม่มีจุดสับสน **รุนแรง** ที่ทำให้ผู้ใช้คิดว่าแอปบันทึกให้อัตโนมัติโดยไม่ตั้งใจ
- [ ] ผู้ทดสอบเข้าใจว่า **ข้อมูลเก็บในเครื่อง** และไม่อัปโหลดออกไป

## 9. จุดตรวจเพิ่มสำหรับ v0.9.0
